# 豆包语音识别是否支持提示词驱动的文本整理

研究日期：2026-07-15

## 结论

豆包语音识别大模型原生提供有限的识别后处理，包括自动标点、数字规整、智能分句和语义顺滑。语义顺滑可删除或修改停顿词、语气词、语义重复词等不流畅内容，因此足以承担一种轻量的“清理模式”。

但截至研究日期，在官方公开的豆包语音识别产品说明、API 能力与更新记录中，没有找到允许调用方提供任意自然语言提示词，并据此重写、重新组织或改变文本风格的公开能力。因此不能把 `context`、热词或语义顺滑等同于通用 LLM 文本整理。

若产品需要“仅校正”“清理口语”“完整重写”及用户自定义提示词，推荐采用两阶段流水线：豆包 ASR 负责转录与可选的内置顺滑，随后由 DeepSeek 或其他文本模型按整理模式处理文本。原文模式可以跳过第二次调用。

## 能力边界

| 能力 | 官方确认 | 在本产品中的用途 |
| --- | --- | --- |
| 自动标点、数字规整、智能分句 | 是 | 提升转录文本可读性 |
| 语义顺滑 | 是 | 删除或修改停顿词、语气词、重复词，做轻量清理 |
| 热词表 | 是 | 提升人名、品牌、术语等特定词汇的识别率 |
| `context` 上下文 | 是 | 给识别提供背景信息、提高语境相关词汇的准确率；不是改写指令 |
| 任意自然语言提示词驱动的清理/重写 | 未在公开官方 ASR 文档中发现 | 应交给独立文本模型 |

“未发现公开支持”不等于证明服务内部绝对不存在相关能力；它表示不能依据当前公开 API 将该能力纳入可实施规格。如果后续控制台或商务开通提供了未公开参数，需要用真实账号和 API 请求另行验证。
